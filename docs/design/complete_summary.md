## 🎯 Key Design Decisions

### 1. Category-Specific Forms
**Why?** Different vendor types have different requirements
- DJ needs: Equipment list, sound system, lighting
- Caterer needs: Cuisines, dietary options, menu planning
- Decorator needs: Styles, materials, sq ft pricing
- Venue needs: Capacity, facilities, multi-day support

**Solution:** Dynamic forms that adapt per category

### 2. Emergency-First UI
**Why?** Vendor cancellations are critical moments for trust
- Immediate notification (red banner, top priority)
- Clear options: Accept replacement or request refund
- 4-5 replacement suggestions auto-matched
- Transparent communication: "We cover price differences"

**Solution:** Dedicated emergency screen, admin dashboard

### 3. Sticky CTA Over Hamburger Menu
**Why?** Key actions (Book Now, Request Quote) should always be visible
- Bottom sticky button on mobile
- Right sidebar on desktop
- High conversion point
- Shows pricing next to action

**Solution:** Implemented in all vendor detail screens

### 4. Tabs Over Page Navigation
**Why?** Vendor detail has multiple content types (packages, items, reviews)
- Desktop: Horizontal tabs (sticky)
- Mobile: Swipeable tabs
- Content loads on-demand (performance)

**Solution:** Shadcn/UI Tabs with custom styling

### 5. Bottom Sheet for Filters
**Why?** Filters shouldn't hide main content on mobile
- Slides up from bottom (90vh max)
- Drag handle to dismiss
- All filters in one place
- "Apply" button at bottom

**Solution:** Gorhom BottomSheet + custom styling

## 🎓 Design Principles Applied

Throughout all 5 documents, these principles guide every decision:

1. **Clarity Over Creativity**
   - Clear information hierarchy
   - Obvious CTAs
   - No ambiguity

2. **Trust Over Friction**
   - Transparent pricing upfront
   - No hidden charges
   - Reassurance messaging

3. **Performance Over Features**
   - Cut scope if it slows down
   - Fast load times matter
   - Mobile-first always

4. **Accessibility Inherent**
   - Not bolted-on
   - WCAG AA standard
   - High color contrast

5. **Mobile as Primary**
   - Design mobile first
   - Responsive is table-stakes
